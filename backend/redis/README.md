# Redis 模块

这个目录提供后端独立的 Redis 基础能力，不绑定具体业务。

## 目录结构

- `client/`: Redis 客户端封装和 `noop` 实现
- `config/`: 配置默认值和校验逻辑
- `shared/`: 通用错误定义和共享 `Options`
- `redis.go`: 模块统一导出入口

当前目标：

- 提供统一的 Redis 配置结构和校验逻辑
- 提供可直接复用的客户端封装
- 在未启用 Redis 时返回 `noop` 实现，方便服务平滑启动
- 为后续接入 `worker`、`rpc` 或其他模块预留统一入口

当前不包含：

- 业务缓存 key 设计
- 路由报价缓存策略
- 图谱同步调度逻辑
- 与上层业务模块的自动 wiring

后续接入时，建议先在上层配置里补 `RedisClient`，再按模块需要注入 `redis.Store`。
